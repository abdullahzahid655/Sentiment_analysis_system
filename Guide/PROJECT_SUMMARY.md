## 🔑 Key Concepts at a Glance

### 1. **Text Preprocessing** (Stage 3)
```
Raw:        "I really don't think this movie is good!!!"
Processed:  "really not think movie good"

Why? Remove noise, expand contractions, lemmatize, remove stopwords
(but KEEP negations like "not" - critical for sentiment!)
```

### 2. **Feature Extraction** (Stage 4)
```
Method 1: Bag of Words (BoW)
  Convert text to word counts: "great movie" → [0,1,1,0,...]
  
Method 2: TF-IDF (Better!)
  Weight words by frequency × rarity
  Rare, informative words get higher weight
  "mediocre" (rare) > "movie" (common)
```

### 3. **Machine Learning Models** (Stage 5)
```
Logistic Regression
  ├─ Assigns weights to each word
  ├─ Fast & interpretable
  └─ ~92% accuracy
  
Naive Bayes (2 variants)
  ├─ Probabilistic approach
  ├─ Assumes word independence
  └─ Good baseline, slightly lower accuracy
```

### 4. **Evaluation Metrics** (Stage 6)
```
Accuracy: Overall correctness (use with caution!)
Precision: Of predicted positives, how many are actually positive?
Recall: Of actual positives, how many did we catch?
F1 Score: Harmonic mean (use when both matter)
ROC-AUC: Overall discrimination ability (0.5=random, 1.0=perfect)
```

### 5. **Deployment** (Stage 8 + FastAPI)
```
Joblib: Save/load ML models to disk
FastAPI: Turn model into REST API
API: Other apps can use your model via HTTP requests
```

## 📈 Model Performance Analysis

### Accuracy vs Other Models
```
Logistic Regression + TF-IDF ........ 92.1% ⭐ BEST
Complement Naive Bayes + TF-IDF .... 89.3%
Multinomial Naive Bayes + TF-IDF ... 88.7%
Logistic Regression + BoW .......... 91.5%
Multinomial Naive Bayes + BoW ...... 87.9%
```

### Why Logistic Regression Wins
1. **Interpretable** - See which words matter
2. **Fast** - Linear complexity, quick training
3. **Effective** - 92%+ accuracy is excellent
4. **Stable** - Converges quickly
5. **Generalizable** - Works well on unseen data

### What's NOT Perfect
- **Sarcasm**: "Oh sure, great movie" → Model might predict positive
- **Mixed sentiment**: "Good plot, terrible acting" → Conflicting signals
- **Domain shift**: Model trained on movie reviews might struggle with product reviews
- **Rare words**: Words not in training data → Unknown sentiment

## 🐛 Troubleshooting

### Q: Model accuracy is lower than expected
**A**: 
1. Check preprocessing - are you removing important signals?
2. Try different hyperparameters
3. Increase training data
4. Use ensemble methods (combine models)

### Q: API returns "Model not loaded"
**A**: Ensure `sentiment_model.pkl` exists in the same directory as `sentiment_api.py`

### Q: Slow predictions
**A**:
1. Use batch_predict endpoint (faster)
2. Add caching for repeated predictions
3. Run on GPU-enabled machine
4. Deploy with multiple worker processes

### Q: Model predicts "positive" for everything
**A**: 
1. Check if data is balanced (50/50)
2. Verify class labels are correct
3. Try class weighting
4. Use different preprocessing

### Q: Preprocessing removes important words
**A**:
1. Keep negations: "not", "never", "no"
2. Keep intensifiers: "very", "really", "extremely"
3. Custom stopword list instead of default
