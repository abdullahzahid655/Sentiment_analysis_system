Text Preprocessing
Clean → Tokenize → Remove stopwords → Lemmatize
NLP Core

Raw text is noisy. We apply a 5-step cleaning pipeline. Key insight: keep negation words like "not", "never", "hardly" — removing them destroys sentiment (e.g., "not good" becomes "good"). Lemmatization is preferred over stemming because it produces real words ("running" → "run", not "runn").

def clean_text(text): text = text.lower() # 1. lowercase text = re.sub(r"<.*?>", " ", text) # 2. remove HTML text = text.replace("n't", " not") # 3. contractions text = re.sub(r"[^a-z\s]", " ", text) # 4. alpha only return re.sub(r"\s+", " ", text).strip() # 5. whitespace def preprocess_text(text): text = clean_text(text) tokens = word_tokenize(text) tokens = [t for t in tokens if t not in STOP_WORDS and len(t) > 2] tokens = [lemmatizer.lemmatize(t, pos="v") for t in tokens] return " ".join(tokens)
4
Feature Extraction
Bag-of-Words vs TF-IDF · Unigrams + Bigrams
ML Core

ML models need numbers, not words. We compare two vectorization strategies. TF-IDF is almost always better — it down-weights words that appear everywhere (like "the") and boosts rare but meaningful words (like "outstanding"). Using bigrams (ngram_range=(1,2)) captures phrases like "not good" and "highly recommend".

# Method 1: Bag of Words (word counts) bow = CountVectorizer(max_features=10000, ngram_range=(1,2)) X_train_bow = bow.fit_transform(X_train) # learn vocab on train X_test_bow = bow.transform(X_test) # apply to test only! # Method 2: TF-IDF (weighted term frequency) tfidf = TfidfVectorizer( max_features=10000, ngram_range=(1, 2), min_df=2, # ignore very rare words sublinear_tf=True, # log smoothing ) X_train_tfidf = tfidf.fit_transform(X_train)
5
Model Training
Logistic Regression · Naive Bayes · Complement NB
ML Core

Logistic Regression learns a weight per word — positive weight = positive indicator, negative weight = negative indicator. It's highly interpretable and excellent for text. Naive Bayes uses probability theory and trains lightning fast. Complement NB is an enhanced variant that handles imbalanced classes better.

# Logistic Regression — best overall for text lr = LogisticRegression(max_iter=1000, C=1.0, random_state=42) lr.fit(X_train_tfidf, y_train) # Naive Bayes — fast probabilistic baseline nb = MultinomialNB(alpha=0.1) nb.fit(X_train_tfidf, y_train) # Compare both on test set print(accuracy_score(y_test, lr.predict(X_test_tfidf))) print(accuracy_score(y_test, nb.predict(X_test_tfidf)))

Evaluation & Metrics
Accuracy · F1 · ROC-AUC · Confusion Matrix
Analysis

Accuracy alone is not enough. Precision measures "of all predicted positives, how many are right?" Recall measures "of all actual positives, how many did we find?" F1-score balances both. ROC-AUC measures discrimination ability. Always use all four in real projects.

from sklearn.metrics import ( accuracy_score, classification_report, roc_auc_score, confusion_matrix ) y_pred = lr.predict(X_test_tfidf) y_prob = lr.predict_proba(X_test_tfidf)[:, 1] print(classification_report(y_test, y_pred, target_names=["Negative", "Positive"])) print(f"ROC-AUC: {roc_auc_score(y_test, y_prob):.4f}")

Production Pipeline
sklearn.Pipeline · inference function · deployment-ready
Advanced

In production, you bundle the vectorizer + model into a single sklearn Pipeline. This prevents data leakage, ensures the vectorizer is always fitted only on training data, and gives you a single object to serialize and deploy via API. Save with joblib.dump().

Core Concepts
🧹
Text Preprocessing
Standardizing raw text by lowercasing, removing HTML, expanding contractions, stripping punctuation, and lemmatizing. Crucial because models can't handle raw noisy text.
🔢
TF-IDF Vectorization
Converts words to numbers. Term Frequency × Inverse Document Frequency rewards words that are meaningful in a document but rare across the corpus. Better than raw counts.
⚖️
Logistic Regression
Learns a positive or negative weight per word. Words like "terrible" get strongly negative weights, "fantastic" strongly positive. Fully interpretable and highly accurate.
📊
Precision vs Recall
Precision = "are my positive predictions right?" Recall = "am I catching all actual positives?" F1-score balances them. Never rely on accuracy alone for real projects.
🚫
Negation Handling
Keep words like "not", "never", "hardly" in your vocabulary. Standard stopword lists remove them — which destroys sentiment. Always customize stopword lists!
📦
sklearn Pipeline
Bundles preprocessing + model into one object. Prevents data leakage, ensures reproducibility, and makes deployment as simple as pickle.dump() or joblib.dump().
